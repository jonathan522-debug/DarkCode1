Mi Camino en el Diseño y Desarrollo Web

Por Jonathan River 25 de Noviembre, 2025 5 min de lectura

Hola, soy Jonathan River, estudiante de Ingeniería en Sistemas y apasionado por el desarrollo y el diseño web. Este blog nace como un espacio para documentar mi aprendizaje, mis proyectos y todo lo que voy descubriendo en el mundo del desarrollo.

Empecé con lo básico: HTML, CSS y algo de JavaScript, pero poco a poco fui metiéndome en tecnologías más modernas como Laravel, React, Vite y arquitecturas por capas. Cada proyecto que hago, ya sea para la universidad o por cuenta propia, me ayuda a entender mejor cómo se construyen aplicaciones profesionales.

Tecnologías que estoy aprendiendo

Estas son algunas de las tecnologías y conceptos en los que estoy trabajando y que quiero dominar para crecer como desarrollador:

  1. Diseño Web Responsive: Crear interfaces que se adapten correctamente a móvil, tablet y escritorio usando Flexbox, Grid y buenas prácticas.
  2. Front con React + Vite: Componentes reutilizables, estados, hooks y una estructura limpia para proyectos reales.
  3. Backend con Laravel: Rutas, controladores, modelos, APIs REST, autenticación y conexión con bases de datos MySQL.
  4. Bases de Datos: Modelado de tablas, relaciones, claves foráneas y consultas eficientes.
  5. Buenas Prácticas: Organización por capas, MVC, código limpio y legible para trabajar mejor en equipo.

Por qué me importa el rendimiento

Algo que he aprendido es que no basta con que una página se vea bonita: también debe ser rápida. Un sistema lento frustra a los usuarios, y en el mundo real eso significa que la gente deja de usar tu aplicación.

"Un buen sistema no solo debe funcionar, debe ser agradable de usar y lo suficientemente rápido como para que el usuario no piense en la tecnología, sino en lo que está haciendo."

— Reflexión personal de Jonathan

Por eso intento aplicar cosas como:

  • Optimizar imágenes para que pesen menos
  • Cargar solo lo necesario (lazy loading)
  • Minimizar y organizar bien el CSS y el JS
  • Usar APIs limpias y bien diseñadas
  • Probar todo antes de subirlo a producción

Herramientas que uso en mi día a día

Estas son algunas de las herramientas que más utilizo mientras aprendo y desarrollo mis proyectos:

  • VS Code: Mi editor principal para Laravel, React y más.
  • Git y GitHub: Para llevar control de versiones de mis proyectos.
  • Chrome DevTools: Para depurar, revisar estilos y analizar rendimiento.
  • Postman / Thunder Client: Para probar APIs creadas con Laravel.

Mi objetivo

Mi meta es convertirme en un desarrollador capaz de crear aplicaciones completas: desde el diseño de la base de datos y el backend, hasta el frontend y la experiencia del usuario. Cada proyecto que hago me deja una lección nueva.

En este blog iré compartiendo avances, problemas que voy resolviendo y pequeñas guías que puedan ayudar a otros estudiantes como yo.
